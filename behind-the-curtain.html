Behind the Curtain

This page shows the process, tools and thinking behind my portfolio website.

Screenshots of my code

Explanation of challenging parts

The most challenging part was implementing the Dark Mode toggle​ and the Background Music player. For the dark mode, I had to learn how to increase CSS specificity​ (using .dark-mode .main-nav) to override the default styles. For the music player, the challenge was syncing the button text with the audio state. I had to write JavaScript logic to check if the music was playing （and change the button text from "Play" to "Pause" ）. These tasks taught me how HTML, CSS, and JavaScript must work together.

Developer tools usage

I have used Live Server to develop the Background Music player. Since audio files require a local server environment to function correctly in a browsers, which means, I have to use a Live Server. When I first tried to open the HTML file directly, the music would not play due to security restrictions. However, once I ran the site through Live Server, the bgm.mp3 file has loaded. Without Live Server providing a stable localhost environment, I would not have been able to successfully implant my favorite music into the website.

Challenges & Reflection

From this project, I learned that planning is more important than coding. At first, I just started coding randomly(when I am learning), which caused many layout issues. Later, I realized that designing the structure first saves time. I also learned the power of separation of concerns: keeping HTML (structure), CSS (style), and JavaScript (behavior) separate. This made debugging much easier. Finally, I learned that small details, like adding the framework and the addition of background music and so on, I also learned that web development is an exercise in patience and creativity, I'm very happy to have received this kind of training also.
